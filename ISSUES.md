## Cross-cutting themes

1. **Every `sudo`-gated step required a real human at a real terminal.** Non-interactive automation cannot supply a password. Plan for this in any future scripted setup.
2. **Small-model (llama3.2 3B) misfires** showed up repeatedly (Loops 20, 32, 34, 37) as reasoning errors or output-formatting noise rather than crashes — these are documented as known limitations per-loop rather than "fixed," since fixing them really means upgrading the model or constraining the task (templates, stricter tool schemas, stricter output parsers), which is a design decision, not a bug patch.
3. **pgvector's Python-side representation as a string** (Loop 9) is worth remembering for any future script that reads embeddings back out of Postgres via psycopg2 without the `pgvector.psycopg2` register-adapter helper.
4. **Hosted-service dependencies (LangSmith, and optionally LangGraph Platform)** cannot be provisioned headlessly — they need a real account and API key supplied by the user. Loops 40-42 were blocked on this until the key was provided; loops needing a *paid/deployment* step beyond just an API key (Loop 42's full Platform deploy) still fall back to local equivalents.
